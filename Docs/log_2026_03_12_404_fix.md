# 2026년 3월 12일 - GitHub Pages 404 에러 원인 파악 및 해결 로그

## 🚨 문제 원인 (Counselor - 상담가 분석)
사용자님이 경험하신 Github Pages 404 오류의 주된 원인은 다음과 같습니다.

1. **Jekyll 빌드 간섭**: GitHub Pages는 기본적으로 Jekyll 엔진을 사용하여 사이트를 빌드합니다. 이때 파일이나 폴더 이름이 `_` (언더스코어)로 시작하면 이를 특수 목적으로 간주하고 무시해버립니다.
2. **Next.js 정적 빌드 구조**: Next.js (특히 `app` 라우터)를 정적 자산(Static HTML)으로 빌드(Export)하면 내부적으로 `_next` 같은 폴더를 생성하여 JS, CSS 등 핵심 에셋을 담아둡니다.
3. **충돌 발생**: 배포된 `.github/workflows/nextjs.yml`을 통해 아티팩트가 업로드될 때, Github Pages의 Jekyll 엔진이 `_next` 폴더를 무시해버려서 화면을 그리기 위한 필수 파일들을 찾지 못해 404 오류가 발생하게 된 것입니다.

## 🛠️ 해결 방안 (Fix - 해결자)
*   **`.nojekyll` 파일 추가**: GitHub Pages에게 "이 사이트(저장소)는 Jekyll 빌드를 거치지 말고 업로드된 파일 그대로 서비스해라"라고 알려주기 위해, 빌드의 기준이 될 루트 경로(로컬 환경의 경우 `public` 폴더 안에 위치 시 빌드 때 `out/` 경로로 복사됨)에 빈 `.nojekyll` 파일을 추가했습니다.

## 📋 다음 단계
이 수정 사항(`public/.nojekyll` 파일 추가 및 커밋)을 깃허브에 반영하고 푸시합니다. 푸시 후 GitHub Actions의 배포 워크플로우가 다시 실행되어 완료되면 정상적으로 페이지가 렌더링될 것입니다.
