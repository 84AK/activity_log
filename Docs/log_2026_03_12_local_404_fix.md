# 2026년 3월 12일 - 로컬 서버 404 에러 원인 파악 및 해결 로그

## 🚨 문제 원인 (Counselor - 상담가 분석)
사용자님이 로컬환경(`npm run dev`)에서 접속할 때 404 오류가 발생한 원인은 `next.config.ts` 파일의 `basePath` 설정 때문이었습니다.

기존 코드:
```typescript
basePath: '/activity_log',
```
GitHub Pages 도메인 구조(`사용자명.github.io/저장소명`)에 맞추기 위해 강제로 `basePath`를 `/activity_log`로 고정해두었습니다. 이 때문에 로컬 서버 실행 시 `http://localhost:3000/`로 접속하면 페이지를 찾지 못하고 (404), `http://localhost:3000/activity_log`로 접속해야만 화면이 나오는 상황이었습니다.

## 🛠️ 해결 방안 (Fix - 해결자)
*   `next.config.ts`에서 환경 변수(`GITHUB_ACTIONS`)를 감지하여 유동적으로 `basePath`를 설정하도록 수정했습니다.
*   로컬 환경에서는 `basePath`가 빈 문자열(`''`)이 되어 `http://localhost:3000/`에서 정상적으로 열리고, 
*   GitHub Actions를 통한 빌드 시에는 자동으로 레포지토리 이름(`/activity_log`)을 감지하여 할당하게 됩니다.

## 📋 다음 단계
이 수정 사항을 깃허브에 푸시하여 안전하게 기록을 남기겠습니다. 터미널의 개발 서버를 껐다가 다시 실행하면(명령어 `npm run dev`) 정상적으로 메인 화면이 렌더링될 것입니다.
