# 2026-04-02 관리자 권한 연동 및 통합 로그인 전환 작업 로그

## 프로젝트 서기(Scribe) 기록 📝

### 1. 작업 개요 (Objective)
- 기존에는 `/admin` 페이지에서 전용 패스워드를 입력하여 관리자로 로그인하는 다소 불안정한 이중 로그인 체계를 사용함. 
- 이를 구글 스프레드시트의 `Users` 시트에 저장되는 데이터베이스 권한 체계와 하나로 통합하고, 홈페이지 로그인 한 번으로 모든 관리자 권한을 부여하도록 시스템을 전면 개편함.

### 2. 세부 구현 사항 (Implementations)
#### 📌 구글 앱스 스크립트 수정 (백엔드)
- `Docs/google_apps_script.js` 내에 `Users` 시트에 `Role` (권한) 컬럼을 추가하도록 개편.
- 메인에서 로그인 시도 시, 입력한 회원 정보가 맞으면 `Role` 값을 참조하여 `"Admin"`(혹은 `"관리자"`, `"admin"`)일 경우 즉각 `role: "admin"`으로 응답하도록 코드를 개선.

#### 📌 프론트엔드 통합 라우팅 및 버튼 추가 (`page.tsx`)
- 관리자 권한(`admin`)으로 로그인이 성공하는 순간, 사용자의 `sessionStorage`에 안전하게 `admin_auth="true"` 플래그와 통신용 비밀번호를 동기화하여 저장.
- 기존 로그아웃 버튼(`renderDashboardView`) 왼쪽에 톱니바퀴 아이콘과 함께 보라색 그라데이션 **[관리자 화면 이동]** 링크 버튼을 배치 (일반 유저에게는 노출되지 않음).

#### 📌 어드민 전용 로그인 페이지 삭제 (`admin/page.tsx`)
- `/admin`으로 직접 타고 들어왔을 때 나타나던 비밀번호 입력 폼 코드를 완전히 제거함.
- `sessionStorage` 기반의 권한 인증을 통과하지 못한 채 접속할 경우 화면 정중앙에 시크한 디자인의 **[접근 통제됨]** 에러 블록을 띄우고 메인 페이지로 되돌아가게 통제 설계 완료.

### 3. 향후 주의 조치 요망 (Action Item) 🔥
- 로컬 `Docs/google_apps_script.js` 코드가 업데이트되었기 때문에, **반드시 실제 구글 스프레드시트 - Apps Script 에디터에 코드를 모두 복붙하고 새 버전으로 다시 배포를 하셔야** 새로 추가된 Role 기능이 동작합니다.
- `Users` 시트 끝에 `Role` 칸이 생성되면, 거기에 사용할 본인 계정에 `Admin` 이라고 적어두고 메인 화면에서 로그인하시기 바랍니다.
