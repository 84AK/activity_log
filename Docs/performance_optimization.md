# 데이터 로딩 성능 최적화 기술 문서

본 문서는 앱의 데이터 로딩 속도를 개선하고 사용자 경험을 극대화하기 위해 적용된 기술적 사항을 기록합니다.

## 1. 개요
기존의 데이터 로딩 방식은 페이지 진입 시마다 서버로부터 전체 데이터를 다시 가져오는 방식이었으며, 이 과정에서 발생하는 네트워크 지연 시간 동안 사용자는 빈 화면이나 로딩 스피너를 기다려야 했습니다. 이를 개선하기 위해 **SWR(Stale-While-Revalidate)** 패턴과 **API 통합(Aggregation)**을 적용하였습니다.

## 2. 적용된 기술 및 변화

### 2.1 SWR (Stale-While-Revalidate) 패턴
- **동작**: 페이지 로드 시 브라우저의 `localStorage`에서 이전에 저장된 데이터를 즉시 불러와 화면에 표시합니다.
- **효과**: 사용자는 '대기 시간 0초'로 이전 데이터에 접근할 수 있습니다.
- **백그라운드 동기화**: 캐시 데이터를 보여주는 동시에 서버에 최신 데이터를 요청하여, 최신 정보가 도착하면 화면을 조용히 업데이트합니다.

### 2.2 API 통합 (Aggregation - Batch Fetch)
- **동작**: '실습 기록'과 '자료실' 데이터를 각각 따로 요청하던 방식에서, 백엔드(`doPost`)의 `getAllData` 액션을 통해 한 번의 요청으로 모든 데이터를 가져오도록 변경하였습니다.
- **효과**: HTTP 요청 횟수가 50% 감소하여 네트워크 부하를 줄이고 데이터 동기화 안정성을 높였습니다.

### 2.3 UI/UX 최적화 (Non-blocking UI)
- **로딩 지연 제거**: 캐시 데이터가 있을 경우 전체 화면을 가리는 로딩 스피너를 제거하였습니다.
- **진행 상태 알림**: 백그라운드 데이터 동기화 중에는 헤더에 `Syncing` 표시를 노출하여 데이터의 신선도를 시각적으로 알려줍니다.

## 3. 코드 변경 사항

### 3.1 Backend (Google Apps Script)
- `Docs/google_apps_script.js`에 `action === "getAllData"` 추가.
- `logs`와 `resources`를 하나의 JSON 구조로 반환.

### 3.2 Frontend (Next.js)
- `src/app/page.tsx` 내 `refreshData` 통합 함수 구현.
- `useEffect` 훅 내 LocalStorage 연동 로직 추가.
- `isRefreshing` 상태를 활용한 헤더 인디케이터 적용.

## 4. 향후 고려 사항
- **캐시 유효 기간 설정**: 현재는 항상 백그라운드 업데이트를 수행합니다. 데이터가 방대해질 경우 TTL(Time To Live)을 설정하여 업데이트 주기를 조절할 수 있습니다.
- **페이지네이션**: 데이터 건수가 많아질 경우(`localStorage` 한계 접근 시) 특정 주차별 데이터만 캐싱하는 전략이 필요합니다.
