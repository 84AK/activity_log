# 2026-04-02 실습 기록 수정 권한 오류 및 UI 개선 완료 로그

## 프로젝트 서기(Scribe) 기록 📝

### 1. 발생한 오류 및 원인 (Error & Bug)
- **에러 상황**: 프론트엔드에서 실습 내용을 변경하고 '수정 완료' 버튼을 눌렀으나, \`ReferenceError: row is not defined\` 안내창이 뜨면서 저장이 실패하는 문제가 발생함.
- **원인 분석**: 
  - 사용자의 요청을 처리하는 데이터베이스 백엔드(Google Apps Script) 상에서 코드가 동작할 때 발생한 버그.
  - 데이터를 수정하기 위해 비밀번호와 관리자 권한을 비교하는 과정\`if (row[headers.indexOf("Password")] !== pass && !isAdmin)\`에서, \`row\`라는 변수가 아예 정의된 적이 없는데 무단으로 가져다 쓰려 하면서 발생한 전형적인 참조 오류(Reference Error)였음.

### 2. 디버깅 및 해결 로직 (Fixing & Resolution)
- 오류가 발생한 \`Docs/google_apps_script.js\` 파일을 열어 코드를 읽어봄으로써, 반복문 내에서 이미 비밀번호 값을 안전하게 빼내 \`actualPass\` 라는 변수에 저장해 둔 것을 확인.
- 해당 참조 오류 지점을 \`if (actualPass !== pass && !isAdmin)\`로 명확하게 대체하여 권한 로직을 완전히 정상화함.

### 3. 그 외 UI/UX 주요 구현 사항 (Implementation Details)
- **카드 리스트 팀명 제거**: 기존의 투박한 '팀 미정' 블루 박스를 아예 삭제하고, 카드를 작성한 '아크' 등 이름의 첫 글자를 보여주는 깔끔한 원형 아이콘으로 대체.
- **점수 항목 구체화**: 팝업 상세 모달 창에서 'Score'를 '실습 만족도 (Score)'로 변경해 점수가 갖는 의미를 보다 명확하게 보완.
- **작성자 전용 수정 버튼**:
  - 조건문 \`(log.author === userName || userRole === 'admin')\`을 세밀하게 삽입하여, 내가 쓴 글이거나 관리자 로그인을 했을 때만 '수정(하기)' 버튼이 노출되도록 구현 완료.
  - 접근성 강화를 위해 상세보기 팝업 하단뿐 아니라 메인 카드 리스트 우측 하단에도 퀵하게 꺼내 쓸 수 있도록 분리 배치.
- **인체 공학적 날짜 달력 분리**: 날짜 입력 시 텍스트 영역을 누르면 키보드로 입력할 수 있게 살려두고, 오른쪽 끝의 달력 아이콘 버튼을 클릭했을 때만 네이티브 DatePicker가 팝업되게 하여 마우스와 키보드 타이핑 사용자를 모두 배려함.

### 4. 주의 조치 요망 (Action Item)
**중요✨**: 로컬(로컬 에디터)의 \`Docs/google_apps_script.js\` 파일은 버그가 완벽하게 잡힌 상태이나, 구글 스프레드시트 쪽에 있는 백엔드에 즉시 자동 배포되지는 않습니다. **반드시 작성된 이 수정본 코드를 전체 복사하여 구글 시트의 앱스 스크립트 화면에 붙여 넣고 '새 배포' 혹은 '배포 관리자에서 버전 갱신'을 해주셔야 정상 동작합니다.**
